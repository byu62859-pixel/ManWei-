## 1. 概述

### 1.1 项目简介

**漫味 (ManWei)** 是一款动漫情感管理平台，为动漫爱好者提供"追番 + 情绪记录 + AI 推荐"一体化体验。

### 1.2 服务端技术栈

| 维度 | 选型 |
| --- | --- |
| 框架 | .NET 8 / ASP.NET Core Web API |
| 数据库 | SQL Server + EF Core 8 |
| 认证 | JWT (HS256) + Bearer Token |
| 外部依赖 | Bangumi API（动漫元数据）、DeepSeek API（AI 推荐） |
| 文档 | Swashbuckle.AspNetCore 6.5.0（已启用） |

### 1.3 客户端

| 客户端 | 说明 |
| --- | --- |
| 微信小程序 | 用户端（追番/打标/AI 助手） |
| PC 管理端 | Vue + Element Plus（管理员后台） |

### 1.4 接口总览

| 模块 | Controller | 接口数 | 鉴权 |
| --- | --- | --- | --- |
| 认证 | AuthController | 3 | 公开 |
| 动漫 | AnimeController | 8 | 公开/登录/Admin |
| 收藏 | FavoritesController | 10 | 登录 |
| 观后感 | ReviewsController | 7 | 登录/Admin |
| 情感标签 | EmotionTagsController | 8 | 登录/Admin |
| 情感曲线 | EmotionCurvesController | 3 | 登录 |
| 用户 | UsersController | 9 | 登录/Admin |
| 数据看板 | DashboardController | 5 | Admin |
| 推荐 | RecommendationsController | 1 | 登录 |
| AI 助手（PC） | PcAiAgentController | 1 | 登录 |
| AI 助手（小程序） | WxAiAgentController | 1 | 登录 |
| AI 助手（Admin） | AiAgentController | 1 | Admin |

**合计：57 个接口**。

## 2. 通用约定

### 2.1 统一响应格式

所有接口（除 NDJSON 流式响应）均使用以下统一包装：

```json
{
  "code": 200,
  "message": "success",
  "data": { /* 业务数据 */ }
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | int | 业务状态码，200=成功 |
| message | string | 提示信息 |
| data | T | 业务数据，类型由接口决定 |

### 2.2 HTTP 状态码使用约定

| 状态码 | 含义 | 触发场景 |
| --- | --- | --- |
| 200 | 成功 | 业务成功 |
| 400 | 请求参数错误 | 校验失败、参数冲突 |
| 401 | 未授权 | JWT 缺失/过期 |
| 403 | 禁止访问 | 角色不足/越权操作 |
| 404 | 资源不存在 | 查不到对应记录 |
| 409 | 资源冲突 | 重复收藏等 |
| 500 | 服务器错误 | 异常未捕获 |
| 503 | 服务不可用 | 第三方服务（Bangumi）失败 |
| 504 | 网关超时 | AI 响应超时 |

### 2.3 鉴权

除明确标注 `AllowAnonymous` 或 `公开` 的接口外，其余接口需要在请求头携带 JWT：

```
Authorization: Bearer <token>
```

Token 来源：登录接口（`POST /api/Auth/Login` 或 `POST /api/Auth/wx-login`）。

### 2.4 时间格式

- 所有时间字段统一为 UTC ISO-8601（如 `2026-07-01T10:30:00Z`）
- 由 `UtcDateTimeConverter` 全局序列化器处理

### 2.5 分页

分页接口统一使用以下 query 参数：

| 参数 | 类型 | 默认 | 说明 |
| --- | --- | --- | --- |
| page | int | 1 | 页码，从 1 开始 |
| pageSize | int | 20 | 每页数量，范围 1-100 |

分页响应统一包装：

```json
{
  "code": 200,
  "data": {
    "items": [ /* 当前页数据 */ ],
    "totalCount": 0,
    "page": 1,
    "pageSize": 20,
    "totalPages": 0
  }
}
```

## 3. 认证 Auth

**Controller**：`AuthController`
**路由前缀**：`/api/Auth`
**鉴权**：全部公开

### 3.1 用户注册

- **Method/Path**：`POST /api/Auth/Register`
- **说明**：注册新用户（用户名 = OpenId）
- **请求体**：

| 字段 | 类型 | 必填 | 校验 |
| --- | --- | --- | --- |
| username | string | ✅ | 3-50 字符 |
| password | string | ✅ | 6-100 字符 |
| nickName | string | ❌ | ≤50 字符 |

- **响应**：`User`
- **状态码**：200 / 400（用户名已存在）

### 3.2 用户登录（密码）

- **Method/Path**：`POST /api/Auth/Login`
- **说明**：用户名密码登录，返回 JWT
- **请求体**：

| 字段 | 类型 | 必填 |
| --- | --- | --- |
| username | string | ✅ |
| password | string | ✅ |

- **响应**：`LoginResponseDto`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| token | string | JWT |
| expiresIn | int | 过期秒数（默认 604800 = 7 天） |
| userId | int | 用户 ID |
| nickName | string | 昵称 |
| role | string | "User" / "Admin" |

- **状态码**：200 / 401（用户名或密码错误 / 账号被禁用）

### 3.3 微信小程序登录

- **Method/Path**：`POST /api/Auth/wx-login`
- **说明**：用 `wx.login()` 获得的 code 换取 openid，自动注册或登录
- **请求体**：

| 字段 | 类型 | 必填 |
| --- | --- | --- |
| code | string | ✅ |

- **响应**：同 `LoginResponseDto`
- **状态码**：200 / 400（code 无效/微信服务失败） / 401（账号被禁用）

## 4. 动漫 Anime

**Controller**：`AnimeController`
**路由前缀**：`/api/Anime`

### 4.1 获取动漫列表（分页+搜索）

- **Method/Path**：`GET /api/Anime`
- **鉴权**：允许匿名
- **Query 参数**（`AnimeQueryDto`）：

| 参数 | 类型 | 必填 | 默认 | 说明 |
| --- | --- | --- | --- | --- |
| page | int | ❌ | 1 | 页码 |
| pageSize | int | ❌ | 20 | 1-100 |
| keyword | string | ❌ | - | 按名称模糊搜索 |
| type | string | ❌ | - | TV / 剧场版 / OVA |
| tagName | string | ❌ | - | 预置标签筛选（首页标签栏） |

- **响应**：`PagedResult<AnimeDto>`
- **排序**：`CreateTime` DESC

### 4.2 获取动漫详情

- **Method/Path**：`GET /api/Anime/{id}`
- **鉴权**：允许匿名
- **路径参数**：`id` - 动漫 ID
- **懒加载逻辑**：若 `BangumiId` 不为空且关键字段（AirDate/Producer/BangumiScore）任一为空，自动调 Bangumi 拉取元数据
- **响应**：`AnimeDto`（含 `Tags` 字段）
- **状态码**：200 / 404

### 4.3 从 Bangumi 同步动漫

- **Method/Path**：`POST /api/Anime/Sync/{bangumiId}`
- **鉴权**：登录
- **路径参数**：`bangumiId` - Bangumi 条目 ID
- **行为**：
  - 若本地已存在 → 更新名称、封面、简介、类型、总集数（仅在 null/0 时覆盖）
  - 若不存在 → 新建动漫 + 标签
- **状态码**：200 / 400（BangumiId 无效 / 获取失败）

### 4.4 批量回填 TotalEpisodes（管理员）

- **Method/Path**：`POST /api/Anime/admin/backfill-episodes`
- **鉴权**：Admin
- **说明**：遍历所有 `BangumiId != null` 且 `TotalEpisodes` 为 null/0 的行，调用 Bangumi 拉取总集数
- **响应**：`BackfillEpisodesResponseDto`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| scanned | int | 候选行数 |
| updated | int | 成功回填行数 |
| skipped | int | BangumiId 为 null 的跳过数 |
| failed | int | 失败行数 |
| updatedAnimeIds | int[] | 成功回填的 Anime.Id 列表 |
| errors | string[] | 最多前 20 条失败原因 |

### 4.5 新增动漫

- **Method/Path**：`POST /api/Anime`
- **鉴权**：登录
- **请求体**：`CreateAnimeDto`

| 字段 | 类型 | 必填 |
| --- | --- | --- |
| name | string | ✅ |
| cover | string | ❌ |
| summary | string | ❌ |
| animeType | string | ❌（默认 "TV"） |

- **响应**：`AnimeDto`

### 4.6 更新动漫

- **Method/Path**：`PUT /api/Anime/{id}`
- **鉴权**：登录
- **路径参数**：`id` - 动漫 ID
- **请求体**：`UpdateAnimeDto`（字段全部可选，仅更新传入的非空字段）
- **状态码**：200 / 404

### 4.7 删除单个动漫

- **Method/Path**：`DELETE /api/Anime/{id}`
- **鉴权**：Admin
- **状态码**：200 / 404

### 4.8 批量删除动漫（管理员）

- **Method/Path**：`POST /api/Anime/admin/batch-delete`
- **鉴权**：Admin
- **请求体**：

```json
{
  "ids": [1, 2, 3]
}
```

| 字段 | 类型 | 必填 | 校验 |
| --- | --- | --- | --- |
| ids | int[] | ✅ | 1-200 个 |

- **响应**：`BatchDeleteAnimeResponseDto`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| requested | int | 请求删除数（去重后） |
| deleted | int | 实际删除数 |
| notFound | int | 不存在数 |
| deletedAnimeIds | int[] | 已删除 ID |
| notFoundAnimeIds | int[] | 不存在 ID |
| errors | string[] | 最多 20 条错误 |
| errorsTruncated | bool | 错误是否被截断 |

- **事务保证**：整批在一个 EF Core 事务里执行，失败整体回滚

## 5. 收藏 Favorites

**Controller**：`FavoritesController`
**路由前缀**：`/api/Favorites`
**鉴权**：全部需要登录

### 5.1 获取收藏列表

- **Method/Path**：`GET /api/Favorites`
- **Query 参数**（`FavoriteQueryDto`）：

| 参数 | 类型 | 必填 | 默认 | 说明 |
| --- | --- | --- | --- | --- |
| page | int | ❌ | 1 | 页码 |
| pageSize | int | ❌ | 20 | 1-100 |
| status | int | ❌ | - | 0=想看 / 1=在看 / 2=看过 |
| tagName | string | ❌ | - | 标签筛选（含用户自定义+预置） |
| orderBy | string | ❌ | - | 见下表 |

**`orderBy` 枚举**：

| 取值 | 排序逻辑 |
| --- | --- |
| `rating_desc` 或 `Rating` | Rating DESC（NULL 排最后） |
| `rating_asc` | Rating ASC（NULL 排最后） |
| `anime_name` 或 `AnimeName` | 动漫名称 ASC |
| _默认_ | UpdateTime DESC |

- **响应**：`PagedResult<FavoriteDto>`

### 5.2 获取收藏状态数量

- **Method/Path**：`GET /api/Favorites/counts`
- **响应**：`FavoriteCountDto`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| all | int | 总数 |
| wish | int | 想看 |
| watching | int | 在看 |
| watched | int | 看过 |

### 5.3 检查是否已收藏

- **Method/Path**：`GET /api/Favorites/check/{animeId}`
- **响应**：`FavoriteCheckDto`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| isFavorited | bool | 是否收藏 |
| favoriteId | int? | 收藏 ID |
| status | int? | 0/1/2 |
| statusText | string? | 想看/在看/看过 |
| progress | int? | 进度 |
| rating | int? | 评分 |

### 5.4 获取收藏详情

- **Method/Path**：`GET /api/Favorites/{id}`
- **路径参数**：`id` - 收藏 ID
- **状态码**：200 / 404

### 5.5 添加收藏（按 AnimeId）

- **Method/Path**：`POST /api/Favorites`
- **请求体**：

```json
{
  "animeId": 1
}
```

- **初始状态**：想看（status=0，progress=0）
- **状态码**：200 / 400（动漫不存在 / 已收藏）

### 5.6 更新收藏

- **Method/Path**：`PUT /api/Favorites/{id}`
- **请求体**（`UpdateFavoriteDto`，字段全部可选）：

| 字段 | 类型 | 校验 |
| --- | --- | --- |
| status | int? | 0/1/2 |
| progress | int? | ≥0 且 ≤总集数（null/0 时按 500 兜底） |
| rating | int? | 1-10；显式传 null=取消评分 |

- **状态码**：200 / 400（参数越界） / 404

### 5.7 删除收藏

- **Method/Path**：`DELETE /api/Favorites/{id}`
- **状态码**：200 / 404

### 5.8 搜索动漫（本地 + Bangumi）

- **Method/Path**：`GET /api/Favorites/search-anime?keyword=xxx`
- **Query 参数**：`keyword` (string)
- **响应**：`AnimeSearchResultDto[]`（最多 15 条）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| animeId | int? | 本地 ID（本地来源时） |
| bangumiId | int | 始终有值 |
| name | string | 名称（中文优先） |
| nameCn | string? | 中文名 |
| cover | string? | 封面图 |
| animeType | string | TV/OVA/WEB/剧场版 |
| source | string | "local" / "bangumi" |

### 5.9 通过 BangumiId 添加收藏

- **Method/Path**：`POST /api/Favorites/add`
- **请求体**（`AddFavoriteByBangumiDto`）：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| animeId | int? | 二选一 | 本地已有动漫 |
| bangumiId | int? | 二选一 | 自动从 Bangumi 同步 |

- **互斥校验**：两个参数必须传一个、且不能同时传
- **状态码**：200 / 400 / 409（已收藏）/ 503（Bangumi 失败）

## 6. 观后感 Reviews

- **响应**：`PagedResult<AdminReviewDto>`（含 UserId/NickName）

### 6.6 管理端：观后感详情（Admin）

- **鉴权**：Admin
- **响应**：`AdminReviewDetailDto`（含完整 Content）

### 6.7 管理端：删除观后感（Admin）

## 8. 情感曲线 EmotionCurves

**鉴权**：需要登录

- **Method/Path**：`GET /api/EmotionCurves/{favoriteId}`
- **路径参数**：`favoriteId`
- **响应**：`EmotionCurveDto[]`（按集数升序）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| episode | int | 集数 |
| emotionLevel | int | 情感等级（1-5） |

## 13. 错误码与状态码

### 13.1 业务错误码（响应体 `code` 字段）

| code | 含义 |
| --- | --- |
| 200 | 成功 |
| 400 | 业务校验失败（如进度越界、必填字段缺失） |
| 401 | 未登录 / Token 无效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 资源冲突 |
| 500 | 服务器异常 |
| 503 | 第三方服务失败 |
| 504 | 网关超时 |

### 13.2 错误响应示例

```json
{
  "code": 400,
  "message": "进度不能超过总集数 25 集",
  "data": null
}
```

### 13.3 鉴权失败

未携带 / 失效的 JWT 会由 ASP.NET Core 中间件返回标准 401 响应（不带业务包装）。
