# PC用户端设计文档

日期: 2026-05-14 项目: AnimeEmotion PC用户端

## 一、项目概述

### 1.1 目标

为普通用户开发PC浏览器端追番应用，对标现有微信小程序功能，并利用PC大屏优势增强数据可视化体验。

### 1.2 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 框架 | React 18 + TypeScript | 主流稳定 |
| UI库 | Ant Design 5.x | 企业级组件库 |
| 状态管理 | Zustand | 轻量易用 |
| 路由 | React Router 6 | 标准方案 |
| HTTP客户端 | Axios | 与现有项目一致 |
| 图表 | ECharts | 数据可视化 |
| 构建工具 | Vite | 快速开发 |
| 目录 | frontend/pc-client | 与pc-admin并列 |

## 二、设计方向约束

### 2.1 核心原则

- **流动布局**: 非固定式，充分利用大屏空间，内容区最大宽度限制在合理范围
- **克制留白**: 呼吸感，不填满，每一处空白都必须 earn its place
- **设计美感**: 精致、有品味，"一个细节做到 120%，其他做到 80%"

### 2.2 视觉基调

安静、克制、不张扬
信息密度适中，留足气口
避免圆角卡片 + 左 border accent 等烂大街组合
避免紫色渐变、emoji 作图标等 AI slop

### 2.3 响应式策略

- 主要目标：1440px+ 宽度
- 内容区最大宽度：1200px
- 侧边栏可折叠
- 大屏下展示更多数据细节，但不堆砌

## 三、功能模块

### 3.1 页面清单

| 页面 | 路由 | 功能 |
|------|------|------|
| 首页 | `/` | 动漫列表、搜索、标签筛选 |
| 动漫详情 | `/anime/:id` | 信息查看、收藏、情感曲线、观后感 |
| 收藏页 | `/favorites` | 分类管理（想看/在看/看过）、进度更新 |
| 观后感 | `/reviews` | 撰写、编辑、删除 |
| 个人中心 | `/profile` | 用户信息 |
| 数据中心 | `/data` | 追番统计 + 情绪分析（重点） |

### 3.2 数据中心（重点页面）

Tab结构：

- **追番统计**: 收藏趋势、评分分布、类型占比、追番时长等
- **情绪分析**: 情感曲线时间轴、全局词云、追番习惯分析

#### 3.2.1 追番统计 Tab

- 收藏趋势折线图（按月/按季度）
- 评分分布柱状图（1-10分各有多少）
- 动漫类型占比饼图
- 状态分布（想看/在看/看过各比例）

#### 3.2.2 情绪分析 Tab（重点打造）

三个视图面板：

**情感曲线时间轴**

- 展示用户追番过程中的情绪波动曲线
- X轴：集数/时间，Y轴：情绪等级（1-5）
- 支持多动漫对比
- 悬停显示集数详情
- 底部可折叠的集数选择器

**全局情绪词云**

- 用户所有情绪标签的分布
- 标签大小 = 使用频率
- 点击标签筛选相关动漫
- 颜色 = 情绪类型

**追番习惯分析**

- 情绪强度与动漫类型的关联
- 如：看战斗番更容易产生高情绪波动
- 以雷达图或堆叠柱图呈现

### 3.3 路由结构

```
/                       → 首页（动漫列表）
/anime/:id              → 动漫详情
/favorites              → 收藏页
/reviews                → 观后感列表
/reviews/:favoriteId    → 观后感编辑
/profile                → 个人中心
/data                   → 数据中心
/data/emotions          → 情绪分析（默认tab）
/data/stats             → 追番统计
```

## 四、API接口对接

### 4.1 认证

| 接口 | 说明 |
|------|------|
| POST /api/auth/login | 登录 |
| POST /api/auth/register | 注册 |

### 4.2 动漫

| 接口 | 说明 |
|------|------|
| GET /api/anime?Page=&PageSize=&Keyword= | 分页搜索 |

### 4.3 收藏

| 接口 | 说明 |
|------|------|
| GET /api/favorites | 获取收藏列表 |
| POST /api/favorites | 添加收藏 |
| PUT /api/favorites/:id | 更新收藏状态/进度/评分 |

### 4.4 情感曲线

| 接口 | 说明 |
|------|------|
| GET /api/emotioncurves/:favoriteId | 获取情感曲线数据 |
| POST /api/emotioncurves | 创建/更新情感记录 |

### 4.5 观后感

| 接口 | 说明 |
|------|------|
| GET /api/reviews | 获取观后感列表 |
| POST /api/reviews | 创建/更新观后感 |
| DELETE /api/reviews/{favoriteId} | 删除观后感（按favoriteId） |

### 4.6 用户

| 接口 | 说明 |
|------|------|
| GET /api/users/me/stats | 获取用户追番统计 |
| PUT /api/users/me/nickname | 修改昵称 |

### 4.7 AI聊天

| 接口 | 说明 |
|------|------|
| POST /api/wxaiagent/chat | 小程序端AI聊天（需普通JWT，角色不限） |

> 注意：现有 WxAiAgentController 支持普通账号密码登录的JWT，PC端可直接复用。但角色定位是"私人追番顾问"，与小程序一致。如需独立的PC端AI助手角色，需新建Controller或修改Service。

## 五、页面布局设计

### 5.1 整体布局

- 顶部导航栏 + 侧边栏（可折叠）
- 内容区：最大宽度 1200px，居中，左右留白
- 无固定侧边栏：导航项横向排列，充分利用垂直空间

### 5.2 首页布局

- 搜索栏居上，大面积留白
- 筛选标签横向排列，紧凑但不拥挤
- 动漫列表采用瀑布流布局：使用 react-masonry-css 实现，列数根据屏幕宽度自适应（1440px+ 建议 4-5 列）
- 卡片高度根据封面比例自然形成，视觉上有节奏感
- **关键：不自己计算列高，必须用 react-masonry-css**

### 5.3 数据中心布局（重点）

- 采用网格+信息层级布局
- 顶部 Tab 切换（追番统计 / 情绪分析）
- 内容区采用 2-3 列网格
- 每个图表模块独立，间距充足
- 图表下方留白，不堆砌文字说明

### 5.4 情绪分析 Tab 详细设计

三个视图区域（可切换）：

**情感曲线时间轴**

- 横向时间轴，主要展示区域
- 支持多动漫对比（不同颜色的线条）
- 悬停显示集数、情绪值、标签
- 底部可折叠的集数选择器

**全局情绪词云**

- 中央大区域展示词云
- 点击标签：侧边弹出相关动漫列表
- 标签大小 = 使用频率，颜色 = 情绪类型

**追番习惯分析**

- 雷达图展示情绪类型与动漫类型的关联
- 或堆叠柱图展示不同类型的情绪分布

## 六、色彩与字体系统

### 6.1 色彩基调

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | #1C1C1E | 深墨色 |
| 背景 | #F7F6F3 | 极淡米白 |
| 强调色 | #D4A574 | 琥珀橙（贯穿全局，如按钮、hover、图表 accent） |
| 主文字 | #1C1C1E | 深墨色 |
| 次文字 | #6B6B6B | 灰色 |
| 边框 | #E8E4DE | - |

> 不使用：紫色渐变、多色聚类

### 6.2 字体系统

- **Display**：衬线体（如 Source Serif、Noto Serif SC）
- **Body**：无衬线体（如 Inter、Noto Sans SC）
- **数据/标签**：等宽字体

```

├── pages/

│   ├── AnimeDetail/      # 动漫详情
│   ├── Favorites/        # 收藏
│   ├── Reviews/          # 观后感
│   ├── Profile/          # 个人中心
│   └── DataCenter/       # 数据中心
├── services/             # API请求
├── stores/               # Zustand stores
└── types/                # TypeScript类型
```

### 7.1 布局组件

- **AppHeader**：顶部导航栏，包含 Logo、导航项、用户信息
- **ContentContainer**：内容区容器，最大宽度 1200px，居中

### 7.2 通用组件

**AnimeCard：动漫卡片**

- 封面图（根据实际比例，不强制正方形）
- 标题（下方，不遮挡封面）
- 评分标签（如果已评分）
- Hover 效果：轻微上浮 + 阴影加深
- **避免**：圆角卡片 + 左 border accent

**SearchBar：搜索栏**

- 大面积留白，不填满
- 简洁的输入框，下方无多余装饰线

### 7.3 数据可视化组件

- **EmotionLineChart**：情感曲线折线图（ECharts）
- **WordCloud**：情绪词云（echarts-wordcloud）
- **RadarChart**：追番习惯雷达图
- **StatCard**：统计卡片（克制，无装饰性图标）

### 7.4 AI 聊天组件

- **ChatBox**：聊天框（简洁气泡样式）
